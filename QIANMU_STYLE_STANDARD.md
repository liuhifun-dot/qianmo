# 千幕 Qianmu Website Style Standard v1.0

## 1. Brand positioning

- 中文名：千幕
- 英文名：Qianmu / Qianmo Lighting（当前域名沿用 qianmolighting.com）
- 品牌句：Light Structure Studio
- 核心定位：照明设计、项目表达、品牌网页、数字内容系统
- 视觉关键词：黑白、结构、留白、克制、项目逻辑、编辑感

## 2. Color system

| Token | Value | Use |
|---|---:|---|
| Black | `#101010` | Main text, buttons, dark blocks |
| Ink | `#1f1f1f` | Body text |
| Dark gray | `#2b2b2b` | Footer background |
| Body gray | `#666666` | Paragraphs |
| Line | `#d6d2ca` | Dividers |
| Paper | `#f7f4ef` | Page background |
| Soft | `#ece8e1` | Secondary background |
| White | `#ffffff` | Cards / reverse text |

## 3. Typography

Primary sans-serif:

```css
Arial, "Helvetica Neue", "PingFang SC", "Microsoft YaHei", sans-serif
```

Accent serif:

```css
Georgia, "Times New Roman", "Noto Serif SC", serif
```

Usage:

- Big brand title: sans-serif, heavy, tight letter spacing
- Editorial section titles: serif, thin, calm
- Navigation and labels: sans-serif, uppercase, high tracking
- Body text: sans-serif, 16–18px, generous line height

## 4. Layout scale

| Item | Standard |
|---|---:|
| Max width | `1480px` |
| Inner content width | `1280px` |
| Page gutter | `22px–72px` responsive |
| Header height | `82px` desktop / `72px` mobile |
| Section spacing | `86px–150px` responsive |
| Large radius | `28px` |
| Card radius | `24px` |
| Button radius | `999px` |

## 5. Homepage order

1. Header
2. Hero: big statement + visual background card
3. Principle quote
4. Services matrix
5. Dark process/showcase section
6. Core values
7. Selected fields / work directions
8. CTA
9. Dark editorial footer

## 6. Component rules

- No green accent.
- Use black/white/paper only.
- Use thin lines instead of heavy borders.
- Do not use fake testimonials or fake client names.
- Use project directions until real project proof is ready.
- Use the same header and footer on all pages.
- Use one `public/assets/css/qianmu.css` stylesheet for all pages.

## 7. Content standard

Every project/case should collect:

1. Project type
2. Location / market
3. Client role
4. Service scope
5. Existing materials
6. Final deliverables
7. Design logic
8. Images / drawings / screenshots
9. Result or decision value
10. Permission status for publication
